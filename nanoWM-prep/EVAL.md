# nanoWM — Evaluation Contract (frozen)

**Rule: this file is finalized before any model code is written.** Numbers drift
when evals are written post-hoc; regressions go invisible. The contract is four
canonical eval calls + a visual contract + a `expected.json` of golden numbers.
Every PR runs `python eval/eval.py --stage N` and is rejected on regression.

This addresses the foundation document's risk #4 (no canonical eval → numbers
drift) directly.

---

## The four canonical eval calls

All four are implemented in a single `eval/eval.py` with a `--stage N` flag.
The output of every call is a dict serialized into `eval/results/stage{N}.json`
and a PNG/GIF in `eval/figures/`. Each call's spec below is **input → output →
pass criterion**.

### Call 1 — k-step latent prediction MSE curve

**Applies to.** Every stage (1–6).
**Input.** 200 held-out trajectories of length 50, fixed seeds (1000–1199).
**Output.** Dict `{k: mse}` for `k ∈ {1, 2, 5, 10, 20, 50}`, normalized by the
encoder's per-dim variance over the eval set. PNG curve plot.
**Pass.** Monotone increasing. `mse[1] < 0.1`. `mse[10] < 5 × mse[1]`.

This is the cheapest predictive eval — no decoder needed, runs in <1 minute on
a 4090, and it's the eval that catches "predictor learned identity" the fastest.

### Call 2 — Pixel PSNR + per-frame FID (reconstructive variants only)

**Applies to.** Stages 1, 2, 3, 6 (anything with a decoder or pixel-space
generator).
**Input.** Same 200 trajectories.
**Output.** Mean PSNR for `k ∈ {1, 5, 10}`. FID computed per-frame against
ground-truth frames at the same `k`. (Per-frame FID, not video FID — `pytorch-
fid` works, video FID is overengineered for the headline.)
**Pass.** k=1 PSNR > 20 dB. k=10 PSNR > 15 dB. FID@k=1 < 50. FID@k=10 < 150.

Numbers are env-dependent; the values above are calibrated for DMC vision.
Calibrate per-stage on day 1 of that stage and update `expected.json`.

### Call 3 — Downstream RL sample-efficiency curve

**Applies to.** Stages 2, 3, 5.
**Input.** Training-time eval, logged every 10k env steps.
**Output.** Episode return vs. env steps, log-x. Compared against the SAC
baseline shipped in `eval/baselines/sac/`.
**Pass.** Curve crosses SAC baseline somewhere; specific crossing point logged
in `expected.json` per env.

For Stage 5 on Crafter: the headline number is **geometric-mean achievement
score at 1M env steps**, calibrated against DreamerV3's reported ~14%.

### Call 4 — Linear / k-NN probe on frozen latents

**Applies to.** Every stage (sanity check that the encoder learned something).
**Input.** A small per-env probe dataset:
- DMC: joint angles (regression, R²).
- Atari: a hand-labeled set of ~1k frames with biome/sprite IDs (classification,
  top-1).
- Crafter: tile-type one-hot at agent position (classification, top-1).
- STL-10 (Stage 4): standard classification top-1.
**Output.** Linear-probe and k-NN-probe metrics.
**Pass.** Both probes > 70% on STL-10 (Stage 4). Other envs: calibrated on
day 1 of each stage.

## The visual contract

**Every stage's README ships a 4-panel rollout GIF**, generated by
`eval/render_panel.py --stage N`:

```
┌─────────────┬─────────────┬─────────────┬─────────────┐
│ ground truth│   1-step    │  10-step    │  50-step    │
│  rollout    │  prediction │ open-loop   │ open-loop   │
└─────────────┴─────────────┴─────────────┴─────────────┘
```

This is the visual that made every world-model paper shareable from 2018 onward.
It is non-negotiable per stage. If you can't render the panel, the stage isn't
done.

## The collapse monitor (Stage 4 + Stage 5)

JEPA's silent failure mode is posterior collapse: the encoder "trains" but
learns an identity-like mapping. Standard losses don't detect this — the
LeJEPA / SALT critique is that JEPA loss is uninformative for model selection.

**Monitor**: every 100 training steps, compute the rank of the embedding
covariance matrix `C = E[(z - mean) (z - mean)^T]` over a batch of 256. Log
`effective_rank = (sum eigenvalues)² / sum(eigenvalues²)` (the participation
ratio). **Alert** if `effective_rank / dim < 0.5`. Logged to the training
PNG curves; an unmonitored JEPA run is a failed JEPA run.

**Eval-time gate.** Per the Stage-4 gate: covariance effective-rank > 0.8 ×
dim on the eval set.

Update protocol: a stage's headline number is set after the **first clean
training run** that meets the stage's gate. Subsequent runs must come within the
stated tolerance or the PR is rejected. This forces the author to either
acknowledge regressions or update the baseline (with a commit message
documenting why).

## CI

A single GitHub Action runs `python eval/eval.py --stage all` on a labeled
self-hosted runner with a GPU on a nightly cron (the lightweight evals: k-step
MSE, PSNR, probes). Full RL training runs are **not** in CI; they're triggered
manually before each release. Nightly CI catches encoder regressions,
collapse, and decoder breakage within 24 h.

If self-hosted GPU isn't worth setting up: a quarterly manual run from the
author's workstation is acceptable, with the results posted as a GitHub
discussion.

---

## What this contract does NOT promise

- Atari-100k IQM. DIAMOND took 2.9 days/game on a 4090; nano-scale runs
  will not match the published numbers, and promising them sets up the repo
  for "this doesn't reproduce" complaints.
- IntPhys 2 numbers. SoTA is near chance; not informative for a teaching repo.
- Minecraft / MineRL anything. Banned per the dependency policy.
- Bit-exact reproducibility across PyTorch versions. We pin `torch`, `numpy`,
  `gymnasium`, but determinism in CUDA convolutions is not promised.

The contract's job is to make the repo **honest, regression-resistant, and
visually legible**. Not to compete on benchmarks.
